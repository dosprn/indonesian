  • UMUM

    • Mengapa program anda tidak berjalan ?

      Saya memerlukan sekitar lebih banyak menjabarkan untuk membantu anda (secara normal, Saya adalah  tidak mampu telepati atau clairvoyance ;). Saya perlu untuk tahu yang persis apa yang tidak dapat dilakukan DOSPRN. . 

      Mohon, gambarkan dengan rinci apa yang sedang anda coba untuk lakukan atas DOSPRN dan apa yang anda melakukan. Jangan melupakan untuk menyebutkan pesan yang anda diterima dari DOSPRN, apakah dokumen yang telah dicetak begitu, mengapa anda tidak suka itu. 

      Sekiranya, beritahulah saya sistem operasi yang anda pakai dan nama printer yang anda gunakan. Saya juga butuh dokumen DOS anda dan/atau isi dari fie job DOSPRN.

      Sebelum mengirim email kesupporing, pastikan anda menggunakan versi terbaru dari DOSPRN dan masalah anda tidak sedang di jelaskan di FAQ ini atau dokumentasi DOSPRN .

    • Apakah anda telah memiliki satu lisensi temporary untuk mencobanya ?

      DOSPRN adalah shareware berfungsi secara penuh. Anda bisa mendownload modul instalasi-dan test semua fitur DOSPRN dengan gratis. Anda tidak perlu satu lisensi KEY untuk tujuan ini. Anda dapat mencoba DOSPRN dan dapat nyaman bahwa DOSPRn dan mejawab permasalahan anda dengan pencetakan dengan DOS sebelum pembelian.

    • Text yang telah dicetak pada kertas termasuk letter dan tanda tanda bahwa itu tidak berada disana. Mengapa ?

      Program DOS anda umumnya menggunakan beberapa Sequence khusus untuk mengatur cetakan. DOSPRN can mengatasi beberapa darinya. Anda harus memilih satu palet (palette) Sequences yang sesuai. Dalam windows propertis DOSPRN, menekan tombol "Sequences" untuk membuka kotak daftar "Sequnces".

      Lihat lagi tentang support sekuen DOSPRN Jika anda adalah pasti anda telah memilih palet(lukis) sesuai urutan, namun demikian dokumen anda meliputi karakter tidak biasa, kemudian 'e-mail' dukungan.

    • Dokumen saya telah dicetak secara tidak benar. Dapatkah saya memberikan document ini untuk dicoba ?

      Buka jendela properti DOSPRN dan uncheck pilihan "Cetak Automatis" . Dengan ini, DOSPRN akan menangkap task cetakan program DOS tetapi tidak mencetaknya. 

      Penangkapan tasking akan diletakkan dalam satu file sementara DOSPRN biasanya (biasanya "%USERPROFILE%\prnspl1"). Berkas (file) ini juga dapat dikirim ke 'e-mail' dukungan.kami Jika anda perlu untuk melanjutkan pencetakan kemudian, adan dapat memilih opsi"Automatis Cetak" lagi. .

    • Saya dapt mencetaka satu file teks dengan DOSPRN opsi "File Cetakan " tetapi laporan dari program DOS saya tidak ditangkap Mengapa ?

      Beberapa program DOs mencoba untuk mengatur printer secara langsung melalui port-port input/aoutput perangkat keras, tanpa menggunakan fungsi-fungsi DOS. DOSPRN tidak bisa menangkap laporan dari beberapa program. Jika aplikasi DOS anda bisa merekam (save) laporan sebagai sebuah file teks, diproses sebagai berikut: bersihkan DOSPRN " Tangkap port" pilih dan set up proprti "job filename" untuk mencocokkan file laporan anda Jangan lupa bahwa file sementara akan di hapus oleh DOSPRN setelah pencetakan. .

  • REGISTRASI

    • Apakah registrasi itu dan mengapa ini diperlukan?

      DOSPRN adan satu produkshareware. Anda dapat mencobanya untuk mendapatkan ide bagaiman fungsi-fungsi ini. cek kompatibilitas dengan program DOS anda dan putuskan jika anda memerlukan ini. Anda dapat mencetak seratus dokument untuk tujuan aini. Pada poin ini bahwa anda menentukan usibility dari DOSPRN, anda dapat membelinya atau uninstall ini. .

    • Apa itu code registrasi dan key registrasi?

      Keamanan key DOSPRN adalah didasarkan pada persedian lisensi individu untuk setiap pemakai. Setelah instalasi pada komputer anda, DOSPRN menggeneret CODE registrasi yang dipertimbangkan konfigurasi hardware dan software anda.

      Buka windows properti DOSPRN dan tekan tombol "Registrasi". Dalam field "Code" anda dapat melihat satu code tergenerate.

      Ketika membeli, anda harus memasukkan code registrasi dari kopyan DOSPRN anda. Supporting kami akan mengirimkan satu key pembuka pada alamat yang dituju. Setelah anda mempaste KEY pembuka pada field dari windows registrasi.

    • Setiap kali saya membuka jendela registrasi yang saya lihat satu kode baru. Apakah ada yang salah?

      Tidak, ini adalah benar. Ini adalah bagian dari sistem keamana key DOSPRN. Anda dapat mengirimkan code yang ada yang adan temukan disana.

    • Jika Saya mengirimkan code registrasi saya, akankan saya meninggalkan windows registrasi dan komputer saya di switch sampai saya menerima key pembuka?

      Anda tidak harus melakukan demikian. Anda dapat mengirimkan kami dari CODE yang anda lihat dalam windows REgistrasi dan tutup DOSPRN atau matikan komputer anda. KEY pembuka anda akan menerima aktivasi lisensi anda. Mohon hati-hati untuk tidak membuat kesalahan dalam mengetik ini dalan CODE atau KEY. Gunakan 'copy' dan 'paste' untuk mencegah salah pengetikan.

    • Setelah mengetik KEY lisensi saya dalam windows registrati, satu pesan "Terima kasi...." muncl di layar. Tetapi DOSPRN masih menampilkan windows yang meminta untuk registrasi setiap saya mencetak laporan.

      Katika kamu mengetik KEY registrasi anda , DOSPRN menerima key apapun dengan satu
      pengecetakan checksum tertentu. Tetapi hanya satu KEY yang valid menghilangkan peringatan ketika anda mencetak laporan-laporan DOS program anda
      .

      Ada beberapa keadaan dapat terjadi
      1) Anda keliru ketiku mengirimakn CODE registrasi atau keteika anda ketik lisensi KEY
      2) Anda mengambil CODE registrasi dari satu komputer yang berberda dimana anda sedang mencoba untuk registrasi DOSPRN.

      Mohon, membuka registrasi DOSPRN jendela saat ini dan mengirim ulang CODE kepada saya.

    • Haruskah saya beli satu lisensi baru jika saya perlu reinstall sistem operasi saya?

      Tidak, lisensi dibeli yang apapun adalah diberikan "[untuk] selamanya". Jika anda reinstall sistem operasi, atau jika anda meng-instal satu versi baru [dari] OS mu pada komputer sama, anda bisa menggunakan KEY registrasi yang telah diterima untuk aktivasi DOSPRN.

      Jika anda membuat satu upgrade penting [dari] komputer anda atau meng-instal DOSPRN di atas komputer baru, kemudian anda harus menghubungi support dan dapatkan satu KEY baru secara gratis.

  • SETTING-SETTING

    • Bagaimana saya mencetak satu teks 130 karakter lebar/luas di atas lembar kertas standar?

      DOSPRN membantu anda mencetak teks dengan ukuran huruf yang ada. Sebagai contoh, jika anda menyiapkan CPI bahwa sepadan dengan 20 terhadap halaman "Huruf-huruf" dalam jendela properti, anda bisa menempatkan lebih dari 130 simbol di atas lembar kertas standar. Anda bisa juga menggunakan Esc-sequences untuk mengubah kepadatan pencetakan di/dalam runtime.

    • Bagaimana saya mencetak satu file DOS?

      Buka windows properti DOSPRN. Tekan "File Cetakan," pilih sebuah file yang mana dapat dicetak dan tekan tombol "Buka". Anda juga dapat mencetak file secara otomatis Untuk melakukan ini copi file anda jadi itu akan mereplace file job DOSPRN Ini biasanya "%USERPROFILE%\PRNSPL1".

    • Bagaimana saya dapat mengset DOSPRN untuk mendukung program DOS dalam bahasa saya?

      Anda pertama harus memilih codepage sesuai. Mohon lihat dalam windows properti DOSPRN ->"Huruf-huruf" tab->area "Setting Huruf"->listbox "Halaman kode emulasi ".

      DOSPRN mendukung codepages sekarang 850 (Eropa Timur), 852 (Eropa Barat), 737 (Yunani), 437 (AS), 862 (Ibrani), 775 (Latvian), 855 (Bulgaria), 1125 (Ukrainian) dan 866 (Cyrillic). 

      Tombol "Preview" mengizinkan anda untuk menampilkan file ASCII.tbl dalam DOS box. Gambar preview harus cocok dengan tampilan dari file ini dalam box DOS. Lihat lagi tentang modul-modul codepages Jika anda memiliki ide jelas dari semua ini, anda dapat membuat modul codepage sendiri (*.cp, *.cpx). Jika tidak, 'e-mail' dukungan.

    • Bagaimana saya menterjemahkan interface DOSPRN kedalam bahasa saya?

      Secara normal, interface bahasa adalah dalam file dengan extensi *.lng. Lihat lagi tentang modul-modul bahasa.

      Anda dapat mencopy English.lng kedalam file BahasaAnd.Ing dan menterjemahkan semua kalimat dalam bahasa anda. Jika anda melakukan ini, mohon kirim kami file bahasa anda dan kami akan memasukkan dalam versi berikutnya dari DOSPRN. 

    • Saya mendapatkan kesalahan dalam interface penerjemahan dalam bahasa saya Bagaimana memperbaikinya ?

      Mohon terima maaf dari kami jika anda menemukan error dalam translasi interface ke dalam bahasa anda. Translasi [adalah] secara parsial dibuat oleh para pemakai kita, dan secara parsial oleh sistem translasi otomatis. Tolong kirimkan kami varian dari translasi anda dimana anda menemukan kekeliruan (atau keseluruhan file BahasaAnda.lng -- file yang telah dikoreksi dari packing interface bahasa).

    • Mengapa tidak tersedia semua huruf-huruf untuk pencetakan dari DOSPRN ?

      DOSPRN menggunakan Unicode untuk display simultan dari simbol codepages yang berbeda. Inilah alasan kenapa huruf dan karakter nasional anda bisa mencetak dengan tepat hanya dengan huruf-huruf dimana mendukung Unicode. Kita merekomendasikan bahwa anda hanya menggunakan huruf-huruf monospaced karena banyak program-DOS menggunakan tabulasi untuk keselarasan teks atau kolom, dan font monospaced yang bisa menampilkan beberapa dokumen dengan baik. Kita merekomendasikan bahwa anda menggunakan huruf Courier New atau Lucida Console.

    • Saya memiliki satu komputer dengan 2 printer. Dapatkan program anda menghandel keduanya ?

      Ya, DOSPRN dapat mencetak pada devais yang
      telah terinstall driver pada komputer anda.

      Tetapi saat ini DOSPRN hanya dapat mencetak satu cetakan pada satu divais pencetakan secara simultan. Anda bisa memilih printer saat ini di jendela properti DOSPRN ("Printer" area, "Pilih" tombol).

      Kami berencana mengembangkan secara simultan dukungan untuk pencetakan pada printer-printer yang berbeda Jika anda memiliki ide bagaimana hal ini harus bekerja, silakan sampaikan email ke kami ide-ide anda.

  • SEQUENCES

    • Apakah sequence (sekuen) itu ?

      Urutan adalah khusus kumpulan kode-ASCII yang digunakan untuk mengelola layout informasi yang sedang ditampilkan oleh satu divais pencetakan. Sebagai contoh, semua printer Epson-compatible  mendukung urutan [dari] standar ESC-P2 (, misalnya. Esc-sequences).

      Urutan memungkinkan mengubah ukuran dan tinggi simbol, kerapatan horisontal dan vertikal dari cetakan, dan memungkinkan mecetak objek grafik.

      Dengan default, DOSPRN mendukung palet urutan sendiri, tetapi itu bisa menandingi lebih penting dan itu sering menggunakan sumber-sumber daya dari standar ESC-P2 Lihat lagi tentang sekuen ESC 

    • Program DOS saya bekerja compatibel dengan printer Epson dan dapat mencetak teks-teks dengan huruf-huruf yang berbeda ukuran Saya telah mencoba DOSPRN, tetapi semua huruf sama pada kertas.

       DOSPRN menggunakan palettenya dari sekuen secara default. Untuk men-switch ke pencetakan dengan dukungan palet ESC-P2, anda harus membuka jendela properti DOSPRN, memilih tab "Utama" dan tekan "Sekuen". Kemudian, pilih "Epson Matrix" dari menu drop-down dan tekan "Terapkan".
