Untuk Pemakai Windows NT/2000/XP

DOSPRN saat ini bisa menggunakan dos-utility prn2file.com ke capture secara otomatis , yang mana aplikasi DOS akan mengirimkan data ini ke printer yang berjalan pada Windows NT/2000/XP.driver ini mengredirect semua cetakan data dalam DOS ke satu file DOS.

Selama instalasi, DOSPRN secara otomatis menempatkan satu panggilan ke prn2file.comdalam file%SystemRoot%\System32\autoexec.ntFile batch ini dijalankan setiap anda menjalankan applikasi DOS. 

Anda juga dapat menjalankan ke prn2file.com secara langsung sebelum program dos anda berjalan. Untuk melakukan ini, buat satu file batch dan panggil prn2file.com dan program dos anda. Tentu, untuk kasus ini anda harus menghapus panggilan dari %SystemRoot%\System32\atuoexec.nt 

Parameter untuk redirector:

PRN2FILE [filename] [#<Num. LPT>] [/S<sizebuf>] [/X<XMSsizebuf>] where filename - name of file to save captured data (default: c:\prnspl) #n - number of LPT (default: #1) /S - conventional memory buffer size in bytes (default: /S4096) /X - XMS buffer size in Kbytes (default: /X200)

Jika PRN2FILE tidak bisa mengalokasikan buffer memori XMS, itu hanya akan menggunakan memori konvensional. Dalam masalah ini size default dari convensional buffer memory akan secara otomatis di perbesar hingg 16kb. .

Jika applikasi dos anda membuat laporan-laporan besar, anda dana menambah parameter ukuran size buffer .

Catatan:

Jika applikasi DOS anda can secara langsung membuat output laporang ke satu file, kami sarankan untuk lakukan ini Dalam masalah ini, anda tidak perlu menggunaka prn2file.com. Masukkan nama file C:\PRNSPL dalam applikasi dos anda sebagai satu path untuk output file dos.

Pada DOSPRN, anda dapat juga mengubah parameter "Job filename" ke file DOS lain yang valid dengan path yang penuh dan khususnya dalam applikasi dos anda .
